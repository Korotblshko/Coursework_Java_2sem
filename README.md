# Custom Thread Pool

Курсовая работа по дисциплине «Многопоточное и асинхронное программирование на Java».

В проекте реализован собственный пул потоков без использования `ThreadPoolExecutor`.


## Структура проекта

Основные классы:

- `CustomExecutor` — интерфейс управления пулом;
- `CustomThreadPool` — основная реализация пула;
- `Worker` — рабочий поток со своей очередью задач;
- `CustomThreadFactory` — фабрика потоков с уникальными именами;
- `RejectedTaskHandler` — интерфейс политики отказа;
- `AbortRejectedTaskHandler` — политика отказа с выбросом `RejectedExecutionException`;
- `TaskWrapper` — обертка над задачей с описанием;
- `Main` — демонстрационная программа.

## Как работает пул

При создании пула сразу запускается базовое количество потоков, равное `corePoolSize`.

Каждый рабочий поток имеет собственную очередь задач. Новые задачи распределяются между очередями по принципу Round Robin. Если выбранная очередь заполнена, пул пытается найти наименее загруженную очередь. Если все очереди заполнены, но количество потоков меньше `maxPoolSize`, создается новый рабочий поток.

Если принять задачу невозможно, применяется политика отказа `AbortRejectedTaskHandler`: задача отклоняется, а вызывающий код получает `RejectedExecutionException`.

Если дополнительный поток простаивает дольше `keepAliveTime`, он завершается. Базовые потоки сохраняются до вызова `shutdown`.

## Логирование

В консоль выводятся основные события:

- создание потоков;
- добавление задач в очередь;
- выполнение задач;
- отклонение задач при перегрузке;
- завершение потоков;
- запуск `shutdown`.

## Демонстрация

В классе `Main` показаны три сценария:

1. Обычное выполнение задач.
2. Выполнение `Callable` и получение результата через `Future`.
3. Перегрузка пула и отклонение части задач.
